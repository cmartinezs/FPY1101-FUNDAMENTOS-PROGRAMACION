# 🧠 Conceptos de programación — vocabulario universal

> **Nivel de entrada:** cualquiera; ideal como apoyo para estudiantes que vienen de PSeInt.  
> **Objetivo:** dominar ideas que aparecen en casi cualquier lenguaje, pero explicadas con **Python como eje principal**.  
> **Lenguaje de referencia para los ejemplos:** Python 3.

---

## ¿Por qué aprender vocabulario antes de aprender sintaxis?

Cuando estudias programación aparecen frases como: *"crea una variable"*, *"la función retorna"*, *"este bloque se ejecuta si la condición es verdadera"*.

Si esas palabras no están claras, aprender Python se vuelve más difícil de lo necesario. Este material ordena ese vocabulario y lo conecta con ejemplos pequeños, reales y fáciles de probar.

> 📌 La idea es que puedas reconocer estos conceptos en Python, relacionarlos con lo que ya conoces en PSeInt y luego transferirlos a otros lenguajes si hace falta.

- 📖 **Definición clara** — qué significa y por qué importa.
- 🌐 **Perspectiva universal** — cómo aparece en otros lenguajes o en PSeInt.
- 🐍 **Ejemplo en Python** — sintaxis real y pequeña.
- ⚠️ **Confusiones frecuentes** — errores típicos de quien recién empieza.

## Resumen de contenidos

### 01 — Vocabulario universal
Reúne los términos base que aparecen una y otra vez cuando aprendes a programar: variable, literal, asignación, bloque, función, parámetro, ámbito, clase, objeto, excepción y más. La idea es que leas Python con menos fricción.

### 02 — Paradigmas de programación
Explica distintas formas de pensar una solución: **imperativa**, **declarativa**, **orientada a objetos** y **funcional**. El foco está en cómo Python puede mezclar estos enfoques en un mismo programa.

### 03 — Estructuras de datos comunes
Presenta arreglos, listas, pilas, colas, diccionarios, conjuntos, árboles y grafos. Se aterriza cada estructura con herramientas reales de Python como `list`, `dict`, `set` y `deque`.

### 04 — Tipos de errores y debugging
Distingue errores de **sintaxis**, **tiempo de ejecución** y **lógica**. Además muestra cómo leer un traceback de Python y cómo depurar con `print()`, el debugger y pruebas simples.

### 05 — Recursividad
Muestra cómo una función puede llamarse a sí misma, por qué necesita caso base y cómo se relaciona esto con la pila de llamadas.

### 06 — Complejidad algorítmica (Big O)
Introduce la idea de medir cuánto crece el costo de un algoritmo cuando aumenta la entrada: **O(1)**, **O(log n)**, **O(n)**, **O(n log n)**, **O(n²)** y más.

### 07 — Principios de buen código
Resume ideas como **DRY**, **KISS**, **YAGNI**, **SRP** y el valor de elegir buenos nombres, reducir duplicación y escribir código fácil de leer.

### 08 — Concurrencia e hilos
Presenta el concepto de tareas que avanzan al mismo tiempo, problemas de sincronización y herramientas comunes para coordinar trabajo concurrente.

### 09 — Entrada y salida (I/O)
Explica cómo un programa lee datos desde teclado, archivos o red y cómo entrega resultados. También distingue operaciones bloqueantes y no bloqueantes.

### 10 — Memoria: Stack, Heap y Garbage Collection
Ayuda a entender qué pasa con las variables, las referencias y los objetos mientras un programa se ejecuta.

### 11 — Patrones de diseño: Creacionales
Reúne patrones centrados en la creación de objetos y en cómo controlar mejor ese proceso.

### 12 — Patrones de diseño: Estructurales
Revisa patrones que ayudan a combinar clases y objetos para formar estructuras más flexibles.

### 13 — Patrones de diseño: Comportamiento
Presenta patrones que organizan la comunicación, el reparto de responsabilidades y la variación de comportamientos dentro del sistema.
